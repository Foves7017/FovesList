# FovesList — AI 使用指南

一个基于**四象限法则**（艾森豪威尔矩阵）的待办事项管理工具。每个事件按重要 / 紧急程度分布在二维坐标图上，一目了然。

## 基础原则

1. **Event 是核心实体**：每个事件有 `title`（唯一）、`importence`（-1~1）、`urgency`（-1~1）、`description`、`sub_event`。
2. **子任务用 `~` 标记完成**：`"写文档"` = 未完成，`"~写文档~"` = 已完成。切换通过 `toggle_sub_event`。
3. **删除 = 归档**：删除的事件进入 `DeletedEvents.json`，可通过回收站恢复。
4. **重名保护**：标题必须唯一，后端会返回 409 阻止重名。

---

## 常用场景

### 用户说"帮我记一下 XX"
→ 调用 `add_event`，询问重要/紧急程度。若用户没指定，根据常识判断：
- 工作汇报、考试复习 → 重要且紧急 (+0.7~+0.9, +0.7~+0.9)
- 学习新技能、健身 → 重要不紧急 (+0.6~+0.8, -0.5~-0.8)
- 取快递、回简单消息 → 紧急不重要 (-0.3~-0.6, +0.5~+0.8)
- 刷视频、发呆 → 不重要不紧急 (-0.5~-0.9, -0.3~-0.9)

### 用户说"我今天做了什么 / 还有什么没做"
→ 调用 `list_events`，按象限汇总。

### 用户说"XX 做完了"
→ 如果 XX 是事件名：调用 `delete_event`（归档）。
→ 如果 XX 是子任务：调用 `toggle_sub_event` 标记完成。

### 用户说"改一下 XX"
→ 调用 `update_event`，只改需要改的属性。

### 用户说"删掉 XX"
→ 调用 `delete_event`。如果有未完成子任务会被拦截，告知用户。

### 用户说"打开待办"
→ 调用 `open_dashboard`。

### 用户说"回收站里有什么 / 恢复 XX"
→ `list_deleted_events` → `restore_event`。

---

## 注意事项

- `sub_event` 参数传 JSON 数组字符串，如 `'["任务A","任务B"]'`。
- 坐标范围是 -1 到 1，建议不要给极端值（如 ±1.0），留点余地。
- 操作前可先 `list_events` 确认事件名无误。
- 如果后端离线，`check_status` 会返回离线提示，提醒用户启动 uvicorn。
- 通常，你需要在添加待办事项后，为用户打开窗口。
